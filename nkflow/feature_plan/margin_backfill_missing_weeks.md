# 信用残高バックフィル — 未取得期間の補完

## 背景

`manual_backfill_margin.py` を使って `margin_trading_weekly` に過去データを追加した。
J-Quants API のレート制限 (429) により、以下の期間が取得できなかった。

## 未取得期間

| 期間 | 週数 (概算) | 状況 |
|------|------------|------|
| 2025-11-30 〜 2026-02-19 | 約 12〜13 週 | 429 エラーで失敗 |

現在 DB に入っている週:

```
2025-09-05, 2025-09-12, 2025-09-19, 2025-09-26,
2025-10-03, 2025-10-10, 2025-10-17, 2025-10-24,
2025-10-31, 2025-11-07, 2025-11-14, 2025-11-21,
2025-11-28,
2026-02-20  ← 元々あったもの
```

## 補完手順

時間をおいてから（J-Quants のレート制限がリセットされた後）以下を実行する:

```bash
cd backend
S3_BUCKET=nkflow-data-268914462689 \
.venv/bin/python scripts/manual_backfill_margin.py \
  --from-date 2025-11-30 --to-date 2026-02-27
```

- 90日以内なので1チャンクで収まるはず
- レート制限が続く場合は時間をさらに空けるか、期間をさらに短く分割する

## 注意事項

- J-Quants の `get_mkt_margin_interest_range` は内部で週ごとにAPIを叩くため、
  1チャンク = 約 90 日 = 約 13 週 ≈ 13 API 呼び出しとなる
- 連続実行するとレート制限にかかりやすい。少なくとも **30 分以上** 間隔を空けること
- スクリプトは冪等 (INSERT OR REPLACE) なので重複実行しても安全

## 完了条件

```sql
SELECT COUNT(DISTINCT week_date) FROM margin_trading_weekly WHERE market_code = 'ALL';
-- 結果が 26 以上になれば信用圧力タイムライン (26週) が正常に表示される
```
