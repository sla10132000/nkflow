## Phase 1: データ前処理・セクター別時系列の構築

### タスク 1-1: セクター分類マッピング

```
JQuantsのlisted_info（または銘柄情報テーブル）から、
各銘柄の「33業種分類」または「17業種分類」を取得してください。

以下のマッピングテーブルを作成:
  銘柄コード → セクター名

もしセクター情報がDBになければ、JQuants APIの
/listed/info から取得するスクリプトを作成してください。
```

### タスク 1-2: セクター別日次リターンの算出

```python
# 以下の処理をPythonスクリプトとして作成してください

1. 株価テーブルから日次終値を取得
2. 各銘柄の日次リターンを算出: r = (close_t / close_{t-1}) - 1
3. セクターごとに時価総額加重平均リターン（または等加重平均）を算出
4. 結果を DataFrame にまとめる:
   - index: date
   - columns: 各セクター名
   - values: 日次リターン

※ 時価総額がDBにない場合は等加重で進めてOK
※ 出来高ゼロ・欠損日は適切にハンドリング

出力: sector_daily_returns.parquet として保存
```

### タスク 1-3: 週次・月次リターンへの変換

```
sector_daily_returns を元に:
1. 週次リターン（金曜締め）
2. 月次リターン（月末締め）
3. 各期間のセクター「順位」（リターン上位→1位）

をそれぞれ算出して保存してください。

出力:
  - sector_weekly_returns.parquet
  - sector_monthly_returns.parquet
  - sector_weekly_ranks.parquet
  - sector_monthly_ranks.parquet
```

## Phase 2: ローテーションパターンの発見

### タスク 2-1: セクター間相関・リードラグ分析

```python
以下の分析を実施してください:

1. セクター間のリターン相関行列（ヒートマップで可視化）
2. ローリング相関（60日窓）の時系列推移
3. リードラグ相関:
   - セクターAのt期リターン vs セクターBのt+1期リターン
   - 全セクターペアで算出 → 「先行指標になりうるセクター」を特定
4. グレンジャー因果検定（pairwise, lag=1〜4週）

結果をヒートマップ・表で可視化し、
「どのセクターがどのセクターに先行するか」をまとめてください。
```

### タスク 2-2: ローテーション状態の定義とクラスタリング

```python
以下の手順でローテーション「状態」を定義してください:

1. 各期間（週 or 月）のセクター順位ベクトルを作成
   例: [金融=1位, 情報通信=2位, ..., 不動産=33位]

2. このベクトルを特徴量として、以下のクラスタリングを実施:
   a. K-Means（k=4〜10で試行、elbow法・silhouetteスコアで最適k決定）
   b. 階層的クラスタリング（Ward法、デンドログラム可視化）
   c. 必要に応じてGaussian Mixture Model

3. 各クラスタの「代表的パターン」を解釈:
   - どのセクターが上位/下位か
   - 景気サイクルとの対応（景気拡大期型、後退期型 etc.）
   - クラスタに名前をつける（例: "ディフェンシブ優位期", "シクリカル相場" 等）

4. クラスタの時系列推移をプロット
   → いつどの状態だったかのタイムラインを可視化
```

### タスク 2-3: 隠れマルコフモデル（HMM）によるレジーム検出

```python
補完的にHMMも試してください:

1. セクターリターンを特徴量として GaussianHMM を学習
   - 状態数: 3〜6で試行（BIC/AICで選択）
2. 各期間のレジーム（状態）を推定
3. 状態遷移確率行列を可視化
4. 各レジームの特徴（どのセクターが強い/弱い）を記述
5. クラスタリング結果との一致度を確認

ライブラリ: hmmlearn
```

## Phase 3: パターン遷移の分析

### タスク 3-1: 遷移確率の算出

```python
Phase 2で定義したパターン（クラスタ or HMMレジーム）について:

1. 遷移確率行列を算出
   - P(次の状態=j | 現在の状態=i) を全ペアで計算
2. 遷移確率のヒートマップを作成
3. 各状態の平均持続期間（何週/何ヶ月続くか）を算出
4. 自己遷移確率（同じ状態が続く確率）をハイライト

5. 2次マルコフ性の検証:
   - P(次=k | 現在=j, 前=i) も算出
   - 1次マルコフとの比較（条件付き確率に有意な差があるか）
```

### タスク 3-2: 外部要因との関連

```python
以下の外部指標との関連を分析してください:

1. マクロ指標（DBにあれば使用、なければスキップ）:
   - 日経平均VIX（恐怖指数）
   - 米国10年金利
   - ドル円為替
   - 景気ウォッチャー調査

2. 各パターン状態における外部指標の平均値・分布を比較
3. 状態遷移の「トリガー」になりうる指標を特定

※ DBに外部指標がなければ、セクターデータのみの内部指標
 （ボラティリティ、セクター間分散、モメンタム等）で代替
```

## Phase 4: 予測モデルの構築

### タスク 4-1: 特徴量エンジニアリング

```python
予測モデル用の特徴量を作成してください:

【基本特徴量】
- 各セクターの過去N期リターン（N=1,2,4,8,12）
- 各セクターの順位（直近1,2,4期）
- セクター間リターンの分散（ローテーション激しさ）
- モメンタム指標（過去4週 vs 過去12週）

【テクニカル特徴量】
- 各セクターのRSI（14期）
- セクターリターンの移動平均乖離率
- ボラティリティ（ローリング標準偏差）

【パターン特徴量】
- 現在のクラスタ/レジーム状態（one-hot）
- 現状態の継続期間
- 直近N期の状態遷移シーケンス

ターゲット変数:
  - 翌期のパターン状態（分類問題）
  - 翌期の各セクター順位（回帰 or ランキング問題）
```

### タスク 4-2: モデル学習と評価

```python
以下のモデルを構築・比較してください:

1. ベースライン:
   - 「現在と同じ状態が続く」予測（遷移確率最大のもの）

2. 機械学習モデル:
   a. LightGBM（分類: 次の状態予測）
   b. Random Forest（同上、比較用）
   c. LSTM / GRU（シーケンス入力で状態予測）

3. 評価方法:
   - 時系列分割のみ（Walk-Forward Validation）
   - 学習: 最初の70% → テスト: 残り30%（さらにローリングで評価）
   - ※ ランダム分割は絶対に使わない

4. 評価指標:
   - Accuracy, F1-score（マクロ平均）
   - 遷移確率ベースラインとの比較
   - 混同行列の可視化

5. 特徴量重要度の分析（LightGBMのfeature importance）
```

### タスク 4-3: 予測結果の実用的な出力

```python
最終的に以下を出力する推論パイプラインを作成してください:

入力: 直近のセクターリターンデータ
出力:
  1. 現在のローテーション状態（パターン名）
  2. 次期の予測状態（確率付き）
     例: "シクリカル相場 → ディフェンシブ転換（確率62%）"
  3. 注目セクター（次期に上昇が見込まれるセクター上位5）
  4. 予測の信頼度（過去のモデル精度に基づく）

出力はJSON形式で保存し、可視化用のダッシュボード（HTML/Streamlit）も作成
```
